# 内容策略：SEO 内容生成

## 1. 适用场景

当用户目标是被传统搜索引擎发现、理解和排名时，使用本模板。不要自动叠加 9day、命理、GEO 或其他垂类规则。

## 2. 核心原则

- **搜索意图优先**：先判断用户是在找定义、比较、方法、清单、案例，还是购买/决策。
- **主题覆盖而非关键词堆叠**：围绕主问题展开相关子问题，让页面完整回答一个搜索任务。
- **E-E-A-T**：体现经验、专业、权威和可信度；不要用空泛形容词冒充可信来源。
- **可读性**：短段落、清晰 H2/H3、结论前置、必要时用表格和步骤。
- **真实叙事**：用具体场景解释抽象概念，避免“随着时代发展”式开头。

## 3. 写作流程

1. 明确目标读者、发布平台、业务目标。
2. 识别主关键词、3-8 个次级关键词、3-6 个真实搜索问题。
3. 判断搜索意图：informational / commercial / navigational / transactional。
4. 选择文章角度：问题解答、避坑、比较、教程、清单、案例复盘。
5. 生成结构：H1、导语、速览、主体 H2/H3、FAQ、结尾 CTA。
6. 自检关键词是否自然出现于标题、首段、至少一个 H2、description 和 FAQ。

## 4. 推荐结构

```markdown
# [包含主关键词的问题式标题]

[具体场景开场，第一段直接回答核心问题。]

## 先说结论

- [一句话结论 1]
- [一句话结论 2]
- [一句话结论 3]

## [H2：用户最关心的问题]

[先回答，再解释。]

## [H2：怎么做 / 怎么判断]

1. [动作]
2. [动作]
3. [动作]

## [H2：常见误区 / 对比]

| 项目 | 正确理解 | 常见误区 |
|---|---|---|

## 常见问题

### [真实搜索问题]
[50-120 字直接回答。]

## 最后

[总结和自然 CTA。]
```

## 5. SEO 自检

- [ ] 标题是否包含主关键词，并且像真人会搜索的问题？
- [ ] 首段是否直接回答问题？
- [ ] 是否覆盖主搜索意图，而不是只解释概念？
- [ ] 是否自然使用主关键词和次级关键词？
- [ ] 是否包含 FAQ 或步骤，增强长尾覆盖？
- [ ] 是否有真实经验、案例、数据、来源或边界说明？
