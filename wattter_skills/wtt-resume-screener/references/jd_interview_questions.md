# JD 引导提问清单

当用户没有提供 JD 文本时，按以下顺序提问。每次只问 2-3 个问题，避免轰炸用户。

## 第 1 轮：定位岗位

1. **这个岗位是招什么人的？**（岗位名称、所在团队、汇报给谁）
2. **是招实习生、校招/应届生，还是已经有工作经验的社招？**
3. **业务背景是什么？**（产品形态、所在行业、目标用户、当前阶段）

## 第 2 轮：核心要求

1. **这个岗位的 3-5 条核心职责是什么？**（招进来要解决什么问题）
2. **必须的硬性条件有哪些？**（学历、工作年限、必须会的技能）
3. **哪些是加分项？**（不要求但有的话优先考虑）

## 第 3 轮：团队与文化

1. **团队规模？候选人进来后会和谁合作？**
2. **公司/团队的当前阶段？**（0-1 / 1-10 / 10-100 / 成熟期）
3. **团队文化更看重什么？**（执行力 / 创新 / 抗压 / 自驱 / 业务敏感）

## 第 4 轮：现实约束

1. **薪资预算范围？**
2. **到岗时间要求？**（急招 / 可等 3 个月 / 不急）
3. **工作地点？是否能接受远程？**
4. **是否有试用期 / 背调要求？**

---

## 何时停止提问

只要满足以下条件即可开始评估：
- 知道岗位级别（实习/校招/初/中/高/专家/管理/总监）
- 知道核心职责（>= 2 条）
- 知道硬性条件（学历、经验、必备技能）
- 知道加分项（可选）

剩余信息可由 AI 根据通用行业经验合理推断，**在报告中标注假设**。

## 推断默认值

| 字段 | 推断策略 |
| --- | --- |
| 学历 | 中级以下默认本科；高级/管理默认本科+；专家/总监默认本科+（不卡硕士） |
| 经验 | 与级别挂钩：intern=在校生、junior=1-3年、mid=3-5年、senior=5-8年、staff=8年+ |
| 薪资 | 根据级别 + 一线/二线城市粗估，并明确标注为"参考值" |
| 软性能力 | 互联网/科技公司默认：抗压、自驱、沟通、协作；传统行业默认：稳定、忠诚、规范 |
