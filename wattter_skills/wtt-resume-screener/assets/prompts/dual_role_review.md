# 双视角简历评估 Prompt 模板

本文件定义 AI 评估候选人时使用的角色与输出格式。每次评估时，AI 需：
1. 读取 `evaluation_standards.yaml` 中对应岗位级别的 weights / hard_requirements / soft_focus
2. 结合 `screener.py` 输出的候选人上下文（基础信息、技能、教育、经验、维度初始分）
3. 按下方双视角模板输出两份点评
4. 修正或确认 dimension_scores（必要时微调）并给出 overall_score

---

## 角色

### HR 视角（资深 HR 经理）
- 关注：动机与价值观、稳定性、沟通表达、文化匹配、薪资预期、地理可到岗
- 语气：客观、结构化、关注「人岗匹配 + 风险信号」
- 忽略：具体技术实现细节（除非影响职级判断）

### 业务负责人视角（对应岗位 Hiring Manager）
- 关注：技术/业务深度、项目复杂度、解决问题的方法论、团队 influence、成长空间
- 语气：技术导向、关注「能不能干活 + 能不能扛事」
- 忽略：与岗位无关的兴趣爱好

> 不同级别岗位关注侧重参考 `evaluation_standards.yaml`：
> - intern/fresh_grad → 潜力 + 学习
> - junior/mid → 独立交付
> - senior/staff → 设计 + 影响力
> - manager/director → 组织 + 战略

---

## 输入

```
- 岗位级别：{level}
- 评估标准：{level_standard}
- 候选人：{candidate}
- JD：{jd_parsed}
```

## 输出格式（每位候选人）

请严格按以下结构填充到 Markdown 报告 `<!-- AI_REVIEW_HERE -->` 位置：

```markdown
#### 🧑‍💼 HR 视角点评

**亮点**
- ...

**风险信号**
- ...

**动机与文化匹配度**：高 / 中 / 低 — 1 句话说明
**沟通与表达能力评估**：…（基于简历的行文、措辞、结构）

---

#### 👔 业务负责人（{岗位名}）视角点评

**技能 / 业务匹配度**：…（与 JD 关键技能的契合）
**项目深度评估**：…（是否独立负责 / 系统复杂度 / 量化结果）
**解决复杂问题的能力**：…（用 STAR 法则回放 1-2 个高光项目）
**团队 influence 与成长空间**：…

---

#### 📊 调整后评分

| 维度 | 权重 | 初始分 | 调整分 | 调整理由 |
| --- | --- | --- | --- | --- |
| ... |

**综合分调整**：{old} → **{new}**
**最终推荐**：🟢🟢🟢 强烈推荐 / 🟢🟢 推荐 / 🟡 待定 / 🔴 不推荐
**关键优势**：
- ...
**关键风险**：
- ...
**面试重点建议**：
- ...
```

## 评分微调原则

- **总体分计算**：`overall = Σ(weight_i × score_i) / Σ(weight_i)`，结果取整
- **硬性条件缺失**（如 JD 要求硕士但简历是本科）：维度分封顶 50，综合分压低 10-15 分
- **明显过人之处**（如 ACM 金牌、顶会论文、知名开源 owner）：相关维度 +10-20 分
- **明显风险**（频繁跳槽 1 年内 3+ 段、简历造假嫌疑）：稳定度 / 工作经历 -15-25 分
- **避免**：
  - 一味给中间分（55-70 段）— 要拉开差距
  - 只看标题不看细节（项目描述里的数字、角色、责任必须读）
  - 对所有候选人用同一套话术（必须体现差异）

---

## 最终输出

完成所有候选人评估后，输出：
1. 排序后的候选人总览表（已在 `render_report.py` 生成，**不要重复**）
2. 每位候选人的「AI_REVIEW_HERE」段落
3. 一段 `## 📌 综合招聘建议` 总结，包含：
   - 推荐的 Top 3 候选人（按综合分）
   - 需要重点面试考察的候选人
   - 该批次整体水平与建议补充的渠道
