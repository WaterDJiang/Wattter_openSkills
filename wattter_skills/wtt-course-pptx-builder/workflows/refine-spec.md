# 规范精修工作流

> 独立的、**可选的** 规范审阅环节。默认流水线会写入 `design_spec.md` + `spec_lock.md` 并自动推进。当用户明确要求精修规范时，策略师先产出完整规范，然后**停止** —— 用户在任何图像生成或 SVG 工作开始之前，可以审阅并修改规范的任意部分（大纲、配色、字体、版式、图像策略、页面节奏……）。

本工作流是**条件性的**，与分阶段模式同形：它从不会自行触发，默认路径保持不变。八项确认阶段先把设计方向定下来，作为抽象的推荐方案；本环节让用户能修订策略师基于这些推荐**实际产出的具体规范**。它对零基础用户尤其有价值——比起前置推荐，他们能更好地判断一份成品规范；而规范的 `§IX` 内容大纲通常是用户最想调整的部分。

## 何时运行

用户**明确要求**在生成之前先精修 / 审阅 / 修订规范。识别以下任一模式：

| 模式 | 示例 |
|---|---|
| `"refine the spec / review the spec first"` | `"produce the spec first, let me review before slides"` |
| `"let me revise the spec, then continue"` | `"send me the spec to confirm, I'll edit it"` |
| 任一"生成前想先检查/迭代设计规范"的请求 | `"draft the full plan, I want to adjust it, then generate"` |

**默认关闭。** 策略师在八项确认打包里以一行短句提示这个选项（见 SKILL.md Step 4）。无请求 → 规范一次性写完、流水线照常自动推进；本工作流永不启动。

**前置条件**：八项确认已落定（mode + visual style + 其余项）。本环节修订八项确认所产出的规范，但不会重新打开确认本身。

---

## Step 1：产出完整规范

完全按照 SKILL.md Step 4 规范跑默认策略师输出：写入 `design_spec.md`（§I–XI）与 `spec_lock.md`。阅读相关 `sources/` 文件，让内容大纲（`§IX`）承载真实事实而不是骨架要点。这里没有特殊处理——这就是普通规范，只是产出的同时已知用户即将审阅它。

---

## Step 2：⛔ 硬停止 —— 呈现、讨论与修订

把产出的规范呈现给用户，**等待用户明确修订或批准后再做任何事**。这是一个仅存在于这条可选路径上的条件性阻塞点；默认流水线"八项确认后自动推进"的纪律不受影响。

用户可以修订**规范的任意部分**，不只是大纲——内容大纲、配色、字体、版式、图标计划、图像策略、页面节奏。用**散文**讨论；不要输出打分量表或逐项评分（机械评分卡违背项目约定）。在合适的时候指出值得再看一眼的地方——但让用户主导。

**参考 —— 讨论切入角度（不是清单或评分）**：用平实语言抛出来提示哪些值得讨论。它们指明一个**方向**，绝不是数字——任何时候都不要把任何一项转成 HEX 值、px 尺寸、比例、页数配额或评分。

- *大纲*：逻辑清晰度（要点是否层层递进）、信息密度（每页体量是否合适——既无灌水也不拥挤）、聚焦（每页是否只讲一件事）、语域（口语化还是正式，与受众匹配）、情感共鸣（开头是否有钩子、结尾是否有回响）、章节均衡（页数预算不偏科）。
- *配色*：方案是否契合内容的情绪与受众，是否有足够的层级与对比度以保证阅读舒适——不纠结于哪个具体 HEX。
- *字体*：标题与正文字体是否形成清晰对比或协调统一，尺寸层级是否清晰易读，字体气质是否匹配视觉风格——不纠结于哪个 px。
- *版式*：结构是否跟随每页的信息权重，还是回退到统一的居中对称网格（即"AI 生成"既视感）。
- *图标 / 图像*：整套图标风格一致；图像为内容服务（hero / 氛围刻意使用）而不是装饰。
- *页面节奏*：`anchor` / `dense` / `breathing` 是否跟随叙事起伏，还是整片平铺的密集。

这些角度与已锁定的 `mode`、visual style 和 §6.1 已经塑造的内容有所重叠——把它们当作讨论切入点，用于浮现值得讨论的东西，而不是另起一轮重做。

**每次修改都要让两个文件保持同步。** 用户批准的任何修订都必须同时落到 `design_spec.md` 和 `spec_lock.md`；出现分歧时以 `spec_lock.md` 为准（见 [`strategist.md`](../references/strategist.md) §6.2）。迭代多少轮都行——只有当用户明确批准规范时循环才结束。

---

## Step 3：交回

用户批准后，`design_spec.md` 与 `spec_lock.md` 同步反映最终修订状态。回到 SKILL.md 正常推进：Step 5（图像获取，若有任意 `ai` / `web` 行）或 Step 6（执行器）。

> 注意：本工作流不复制策略师内容。它只在规范产出与流水线其余部分之间插入一个"审阅与修订"的检查点。`strategist.md` / SKILL.md 仍是规范如何撰写的权威依据。
